# 木曜セミパーソナル枠 本番反映チェックリスト

## 前提

この不具合は、コード側の固定スケジュール生成ロジックと、Supabase に過去生成済みの木曜セミパーソナル枠が残っている可能性の両方が原因になり得ます。

そのため、コード反映だけでは完全解消しない場合があります。Vercel 本番反映後に、必要に応じて本番 Supabase の既存データをクリーンアップしてください。

## 修正内容

- 木曜日は、祝日判定より先にセミパーソナルを空配列にします。
- これにより、木曜祝日を含めてセミパーソナル枠は生成されません。
- 固定枠同期時に、将来の木曜セミパーソナル枠を検出します。
  - 予約なしの枠は削除します。
  - 予約済みの枠は履歴を壊さないため `is_open = false` にします。
- 初期サンプルデータに残っていた木曜セミパーソナル枠は月曜に変更しました。
- 本番確認用 SQL として `supabase/production_remove_thursday_semi_personal_slots.sql` を追加しました。

## 本番反映手順

1. `main` に最新コードが反映されていることを確認します。
2. Vercel の Production Deployment が成功していることを確認します。
3. 本番 Supabase SQL Editor で `supabase/production_remove_thursday_semi_personal_slots.sql` を開きます。
4. SQL ファイルの「1) 事前確認 SELECT」だけを先に実行します。
5. 表示された対象が、将来の木曜セミパーソナル枠だけであることを確認します。
6. 問題なければ、DELETE / UPDATE を実行します。
7. 「4) 事後確認 SELECT」を実行し、将来の木曜セミパーソナル枠が残っていないことを確認します。
8. Production URL でセルフジム予約画面を開き、木曜日を確認します。

## 実画面で確認すること

- 木曜日の「その日の予約不可時間」に `20:15〜21:55 セミパーソナル` が出ていないこと。
- 木曜日の `20:45` 前後が、他の予約・通常ヨガ・個別ブロック等と重なっていなければ空き候補として表示されること。
- 木曜日の通常ヨガは、必要な時間帯だけ表示されること。
- 他曜日のセミパーソナル固定枠には影響が出ていないこと。

## 注意

`production_remove_thursday_semi_personal_slots.sql` は、最初からファイル全体をまとめて実行しないでください。

必ず、事前確認 SELECT → 対象確認 → DELETE / UPDATE → 事後確認 SELECT の順で実行してください。
